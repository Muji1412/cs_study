# 📚 어떻게 공부해야 할까? (학습 가이드)

로드맵이 있어도 **"어떻게"** 공부하느냐에 따라 효율은 천지차이입니다.
비전공자이고, 재취업이 목표라면 **"학문적인 연구"가 아니라 "공학적인 해결 능력"**을 보여주는 것이 핵심입니다.

다음 원칙들을 꼭 지켜주세요.

공부한 내용을 머릿속에만 두지 마세요. 휘발됩니다.
Github, Notion, Blog 어디든 좋습니다. **"남에게 설명하듯이"** 정리하세요.

*   **나쁜 예:** "스택은 LIFO 구조이다." (교과서 복사)
*   **좋은 예:** "스택은 접시 쌓기와 같다. 웹 브라우저의 '뒤로 가기' 버튼이 바로 이 스택을 활용한 예시다. Java에서는 `Stack` 클래스보다 `Deque`를 쓰는 것이 좋다고 한다. 왜냐하면..."

> **Tip:** 면접관은 당신의 블로그를 보고 질문할 확률이 높습니다. 내가 자신 있게 대답할 수 있는 내용으로 채우세요.

## 2. 💻 손으로 익히기 (백문이 불여일타)

CS 지식은 눈으로 읽으면 다 아는 것 같지만, 막상 코드로 짜려면 손이 안 움직입니다.

*   **자료구조:** 직접 클래스를 만들어보세요. (`MyStack`, `MyQueue`, `MyLinkedList`...)
*   **운영체제/네트워크:** 관련 명령어나 코드를 실행해보세요.
    *   쓰레드 문맥 교환(Context Switching) 확인해보기
    *   Wireshark 같은 툴로 패킷 직접 뜯어보기
    *   Java Socket 프로그래밍으로 간단한 채팅 서버 만들어보기

## 3. 🗣️ 면접 시뮬레이션: "키워드" 중심으로 말하기

면접은 대화입니다. 질문을 받았을 때 **두괄식**으로 핵심 키워드를 던지고, 부연 설명하는 연습을 하세요.

**연습 방법:**
1.  주제 리스트를 랜덤으로 뽑습니다. (예: "해시맵", "가비지 컬렉터")
2.  아무것도 보지 않고 1분 동안 설명해봅니다. (녹음 필수!)
3.  녹음을 들어보고, 버벅거리는 부분이나 틀린 내용을 체크합니다.

> **예시:**
> **Q: GC가 무엇인가요?**
> **A:** (핵심) 자바 힙 메모리에서 사용하지 않는 객체를 찾아 삭제하는 프로세스입니다.
> (부연) 개발자가 메모리를 직접 관리하지 않아도 되어 편리하지만, 실행 시 Stop-the-world가 발생하여 성능에 영향을 줄 수 있습니다. 따라서 적절한 튜닝이나 GC 알고리즘 선택이 중요합니다.

## 5. 멘탈 관리

*   **"남들은 다 아는데 나만 모르나?"** -> **네, 모를 수 있습니다.** 원래 CS는 방대합니다. 전공자들도 졸업할 때 다 까먹고 다시 공부합니다.
*   **"이걸 언제 다 하지?"** -> **하나씩 패면 됩니다.** 오늘은 '프로세스' 놈만 팬다, 내일은 '쓰레드' 놈만 팬다. 이렇게 생각하세요.
*   **이미 실무 경험이 있다는 것**을 잊지 마세요. 이론을 배우면 "아, 그게 이거였어?" 하는 유레카 모멘트가 반드시 옵니다. 그때가 실력이 퀀텀 점프하는 시기입니다.

포기하지 않으면, 반드시 닿습니다. 화이팅! 💪
